Introduction

JavaScript is a cross-platform, object-oriented scripting language used to make webpages interactive (examples: animations, buttons, popup menus, etc.). JavaScript can be connected to the objects of its environment to provide programmatic control over them.

JavaScript libraries contains standand list of objects, such as Array, Date, and Math, operators, control structures and statements. Core JavaScript can be extended for a variety of purposes by supplementing it with additional objects; for example:

  • Client-Side JavaScript extends the core language by supplying objects to control a browser and its Document Object Model (DOM). Examples of Client-Side extensions allows an application to place elements on an HTML form and respond to user events such as mouse clicks, form inputs, and page navigations.
  • Server-Side JavaScript extends the core language by supplying objects relevant to running JavaScripts on a server. Example includes, server-side extensions allow an applicatoin to communicate with a database, which provides continuity of information from one invocation to antother of the application, or it performs file manipulations on a server.

Java and JavaScript

JavaScript and Java are similar in some ways but fundamentally different in some others. The JavaScript language resembles Java but does not hava Java's static typing and strong type checking. JavaScript follows most Java expression syntax, naming conventions and basic control-flow constructs which was the reason why it was renamed from the LiveScript to JavaScript.

JavaScript is a very free-form language compared to Java. You do not have to declare all variables, classes, and methods. You do not have to be concerned with whether methods are public, private, or protected, and you do not have to implement interfaces. Variables, parameters, and function return types are not explicitly typed.

Java is a class-based programming lanugage designed for fast execution and type safety. Type safety means, for example, that you can't cast a Java integer into a object reference or access it private memory by corrupting the Java bytecode. Java's class-based model means that programs consist exclusively of classes and their methods. Java's class inheritance and strong typing generaly require tightly coupled object hierarchies. These requirements make Java programming more complex than JavaScript programming.

  • JavaScript
        • Object-oriented. No distinction between types of objects. Inheritance is through the prototype mechanism, and properties and methods can be added to any object dynamically.
        Variable data types are not declared (dynamic typing, loosely typed).
        Cannot automatically write to hard disk
  • Java
        • Class-based. Objects are divided into classes and instances with all inheritance through the class hierarchy. Classes and instances cannot have properties or methods added dynamically.
        Variable data types must be declared (static typing, strongly typed).
        Can automatically write to hard disk

Variables

Variables are containers for storing data (storing data values).

In this example, x, y, and z, are variables, declared with the var keyword:

var x = 5;

var y = 15;

var z = x + y;

Variable Scope

JavaScript has two different variable scopes: global and local. Any variable that is declared outside of a function can be releated to the global scope, and it's accessible from everywhere in the code. Every function should have its scope, and any variable declared in that function can be accessible from that function only, or nested functions. As in JavaScript, the local scope is also made by functions, we can also call it a function scope. Whenever you put one functin inside another function, you create a nested scope.

Example of Variable Scope:

{
// work with the local variables that must not be seen outside
let showWelcome = "Welcome to the Technical Documentation Page"; // only visible in this block
console.log(showWelcome); // Shows Welcome Message
}
console.log(showWelcome); // Error: showWelcome is not defined

Global Scope

A variable declared outside a function, is called a Global Scope. Example:

let name = "Joe";
// code here can use name

function myFunction() {
// code here can also use name
}

Constants

Constants can work in almost all browsers. Constants should be in all Capitals. To declare a Constant:

const CONSTANT_VALUE = 200;

To assign a new value:

console.log(CONSTANT_VALUE);

If...Else Statement

Use the

else

statement to specify a block of code to be executed if the condition is false.

if (condition) {
// block of code to be executed if the condition is true
} else {
// block of code to be executed if the condition is false
}

While Statement

The

while

loop loops through a block of code as long as a specified condition is true. Example:

whie (condition) {
// code block to be executed
}

Function Declarations

JavaScript functions are defined with the function keyword.

You can use a function declaration or a function expression.

Examples:

function functionName(parameters) {
// code to be executed
}

function myFunction(a, b) {
return a * b;
}
